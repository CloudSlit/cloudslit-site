What is CloudSlit?

CloudSlit 旨在构建一个全球web3去中心化的数据私人检索系统，构建具有隐私保证、交互式、低延迟通信的网络系统， CloudSlit 让用户保留隐私的情况下，能够让用户拥有交互式的联网和开展活动权限。 CloudSlit 凭据加密并嵌入（包括私有数据的零知识证明在内的）特定服务所需的数据。 这些凭据的验证方式是去中心化的、公开的，不会泄露用户与其想要访问服务之间的任何链接。 CloudSlit 帮助用户重新获得web2下被巨头侵蚀的隐私安全信息，使当前全球热的零信任网络技术结合web3更好地帮助用户掌握 自己的安全隐私数据，并为用户提供良好的零信任安全网络产品和平台体验。

动机

目前可用于具有隐私保证、交互式、低延迟通信的系统非常有限，而且构建具有这三个特性的系统非常困难。 迄今为止开发的解决方案侧重于传统的单一可信来源数据发布者的 Web 模型（与可以提前复制数据的内容寻址系统相反）， 并且在其招致的延迟和威胁模型方面都存在缺陷。

我们针对 Protocol Labs RFP-014 文件，提出我们独特的解决方案，开发出可部署的原型。

现状

最广泛部署的读者隐私方法依赖于混合网络和类似混合网络的多跳系统（Nym、Tor）或可信硬件（SGX）。

这些解决方案中的每一个都需要权衡取舍。Tor 和 Nym 会产生更高的延迟成本。 这些系统使用的额外网络跃点使它们很难在速度上与 web2 CDN 竞争。 今天的多跳系统还没有考虑如何优化具有自我认证、可缓存内容的非 Web 案例的性能。

SGX 等硬件解决方案已在许多场景中用作隐私解决方案，尤其是 Signal。 这种抽象提供了低延迟，但依赖于对集中式服务器运营商和硬件制造商的信任。

学术文献中已经开发了对安全/隐私平衡的其他潜在改进，但尚未跨越创新鸿沟进入生产。 其中包括像全同态加密（FHE）这样的密码系统，DARPA 的 DPRIVE 计划的核心， 以及像PIR-Tor这样的大规模可扩展匿名通信系统。从盲目令牌到零知识证明和统计数据结构的 有前途的技术也可能在实际系统中发挥作用。

特性

隐私性

CloudSlit 是以零知识证明和did机制为基础的网络，为防止网络层元数据的泄露和收割提供了强有力的保证。 它是一个通用的隐私覆盖网络，具备不可知性，可以接驳几乎其他任何数字应用程序或服务， 使个人和数字服务提供商能够保证自己或用户的隐私。

低延迟

利用类混合网络激励竞争关系，用户来选择使用的节点，将主动权会归于客户，实现去中心化的单跳系统， 既保证了隐私性，又保证了延迟低的特性。

交互式

CloudSlit 节点运营方获得奖励的依据是稳定性证明，确保所有人的隐私。 CloudSlit 仅使用能满足需求的服务稳定性证明的数据包，尽量减少不必要的环保成本。 节点运行方之间存在竞争关系，用户可自由选择使用的节点，提供便利的交互性。

网络质量

网络质量验证器可以保证网络提供者提供良好的网络服务环境，支撑服务的良性发展。

组件设计

任何人都可以运行一个完整的节点，该节点承载分散网络的元数据，并提供元数据联网和事务匹配平台。

对于所有用户和Dao的数据，我们使用Filecoin的web3.storage分散存储用户数据。

我们的节点通过libp2p kademlia DHT和IPFS网络通过对等点发现和路由实现自动联网， 并通过libp2p的PubSub功能实现多个节点之间的数据同步。

我们使用智能合约。提供安全的交易流程和安全的交易环境。

客户端软件用户连接到提供商以建立零信任网络安全隧道。
